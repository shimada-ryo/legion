## a02 quality notes (carry to a03/a04)

From the final cross-cutting review:

- **`approvalIdToSessionId` and `adapters` Maps grow forever.** No cleanup on workflow completion. Latent memory leak for long-running servers. Fix candidate: `delete()` from both maps when a `status_change(completed|failed)` event flows through the `onAny` tap. Worth doing in a03 if convenient, otherwise a04.
- **WS subscribe-first defends against a non-existent race today.** `EventLog.append → notify` and `bun:sqlite`'s `db.query().all()` are both synchronous, so the buffer window is currently zero. The mechanism is correct and tested; the complexity earns its keep only if a future async refactor introduces real I/O between `historyWithSeq` and `tail`. Consider a one-line comment if you touch that file.
- **`tasks/inbox/subscriptions: []` are deliberately empty** in `handleInstanceDetail`. Don't add tests asserting they are non-empty; future phases will populate.
- **`history()` and `historyWithSeq()` share SQL** in `reader.ts`. Code-review accepted the duplication on hot-path grounds (4 callers of `history()` would pay an extra allocation per row if DRYed). If a future change touches the SQL, DRY them at that point.
- **Test runner gotcha**: running root-level `bun test` (without `run`) loads web .tsx tests under a Bun runtime that lacks DOM, producing 4 spurious `document is not defined` failures. Always use **`bun run test`** (workspace runner via `bun run --filter='*' test`), which yields the actual 160/1/0.
- **100ms `setTimeout` pattern is proliferating in server tests** (15 occurrences across 5 files). Consider extracting an `awaitWorkflowCompletion(server, wfId)` helper in `packages/server/test/helpers/` before a03 adds more.

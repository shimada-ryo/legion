## What landed in the 2026-05-14 smoke session

Three critical bugs (each a "spec said X, implementation did Y" gap, none caught by the existing test suite) were fixed:

- **I-6: worktree isolation was broken** (`aebf4e8`). The SDK option is `cwd`, not `workingDirectory`. Since Phase 1 merged, every agent had been operating in the user's repo root via `process.cwd()` fallback instead of its assigned worktree. Worktrees were created but never written to.
- **I-7: Vite dev proxy hijacked SPA navigation** (`8ca116f`). Browser navigation to `/instances/:id` was proxied to the API server, returning JSON. Namespaced all backend routes (HTTP + WS) under `/api/*` so the proxy rule no longer collides with React Router routes.
- **I-8: approval flow used the wrong SDK mechanism** (`84bd004`). The SDK's interactive permission path is the `canUseTool` callback, not the `PreToolUse` hook. legion's `PreToolUse` hook never fired for the "ask user" flow, so the yellow Approve/Deny card never appeared. Switched to `canUseTool`.

Two UX / docs improvements:

- **preflight checks in `start.ts`** (`57052f5`) — actionable messages for missing `.git`, no commits, missing `workflows/`.
- **handoff doc updates** (`45a1e18`, `86fcce9`) — OAuth auth path (`CLAUDE_CODE_OAUTH_TOKEN` for subscription users), `/api` curl examples, Windows-specific guidance (SIGINT through `bun --filter` is unreliable; use direct invocation; kill-by-port snippet).

One issue remains open: **I-9 — Implementer never commits to its branch.** The initial prompt does not instruct the agent to `git commit` after editing. The worktree's working tree carries the changes, but the branch HEAD remains at the base commit, so `git diff <base>..<branch>` is always empty and the Diff tab shows `(no changes)`. This is a prompt / role-profile design issue, not an adapter bug — likely best resolved when Phase 2 reworks role definitions.

## Phase 2 enablement concerns (unchanged from prior handoff)

The current shape works for Phase 1 single-role spawning but has known seams for Phase 2:

1. **`ctx.adapters` map keyed by `workflowInstanceId`** — fine for 1 workflow = 1 agent. Phase 2's Director–Worker means many agents per workflow; re-key by `sessionId` so `POST /api/instances/:id/approvals/:approvalId` resolves to the right adapter.
2. **`WorkflowInstance.agentInstances: []` is always empty.** Need an `agent_instances` SQLite table joined to workflow_instances, populated as `triggerWorkflow` (and Phase 2's delegate spawn) creates agents.
3. **`worktreeAdd` only accepts a commit SHA**, not a branch ref. Phase 2 Reviewer worktrees should `--detach <branch>` to see the latest Implementer commit.
4. **`triggerWorkflow` spawns only the role at the trigger.** Phase 2's `delegates` edge needs a real graph walk.
5. **`CanvasOverlay.deriveActiveRoles`** highlights all role nodes once any event arrives. Phase 2 needs a `sessionId → roleNodeId` map (depends on #2).

## Project in one paragraph

**legion** is a coding-agent control plane for individual developers. The vision: "ひとりの開発者が AI を使って企業並みの開発力を発揮する". It orchestrates multiple coding agents (Claude Code first; Codex, Gemini next) as a coordinated team with roles (Director / Implementer / Reviewer / Tester / Knowledge Keeper). Agents communicate through a **Blackboard substrate** that unifies local (same machine) and remote (across machines) coordination. The Web UI uses a **two-layer model**: Layer 1 is the team topology (designed as YAML, edited visually); Layer 2 is the live execution overlay.

## Critical decisions to internalize (summary of D-001〜D-020)

Full text in the minutes. Highlights:

- **D-001**: Individual developer target. Local-first, single-tenant. No RBAC priority.
- **D-002**: Git worktree first-class. Director–Worker pattern. Control Plane (long-running Bun server) ≠ Agent Runtime (spawned subprocesses).
- **D-003**: SQLite for Blackboard / event log to start. Migrate later if needed.
- **D-005**: TypeScript + Bun. No fallback to npm/pnpm.
- **D-008**: React + React Flow + Vite. **No TanStack Query** (security incident on 2026-05-11; even if resolved, defer). Use `fetch` + React state hooks + WebSocket; consider SWR later if HTTP query caching becomes needed.
- **D-009**: Visual Workflow Editor is V1 required, but implemented as **"UI先行モック + runtime 並走"** — NOT pure UI-first. Phase 1 has Track A (headless runtime) and Track B (workflow editor static mockup) running in parallel; Phase 2 connects them. Risk R-001 captures the齟齬 concern.
- **D-010**: Supply chain monitoring policy. Treat seriously.
- **D-012〜D-020**: Two-layer model for the workflow editor.
  - **D-012**: Layer 1 (Topology, static, editable) + Layer 2 (Execution, dynamic, overlay).
  - **D-013**: Layer 1 block = Role; Layer 2 block = Agent Instance; sub-blocks = Tasks.
  - **D-014**: Tasks are encapsulated within an Agent Instance. Cross-instance coordination is ONLY via Layer 1 edges (delegates/publishes/subscribes/reviews/synthesizes/triggers). Each Instance has an internal reactor that absorbs external events and creates intra-instance tasks.
  - **D-015**: Multiple edge types (triggers / delegates / publishes / subscribes / reviews / synthesizes).
  - **D-016**: Blackboard nodes are meta-blocks present in both layers.
  - **D-017**: Human-gate nodes are first-class.
  - **D-018**: Workflow Template vs Workflow Instance separation (BPMN/Temporal-style).
  - **D-019**: Templates stored as YAML in `workflows/`, DB cache. Git-versioned.
  - **D-020**: Role lifetime property: `per-task | per-workflow | persistent`.

### Subagent-driven dispatch pattern observed in session 3

Three subagent dispatches per task is optimal:
1. **Implementer** — gets the full task text from the plan + project context + report expectations.
2. **Combined reviewer** — does spec compliance review THEN code quality review in one subagent. This saves ~30% tokens vs separate dispatches and the review output is still thorough.

Token cost per task ranged ~70K (small type-only) to ~150K (large integration tasks like a01 Task 11/12). Plan accordingly for a02/a03/a04 — combined remaining work ~15 tasks = ~1.5M tokens total.
